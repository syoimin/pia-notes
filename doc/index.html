🔄 動作フロー

1. 起動フェーズ

ラズパイで node server.js 実行 → HTTPサーバー(3000) + WebSocketサーバー(8080) が起動

2. 接続フェーズ
  • 右手タブレット: http://192.168.1.100:3000/melody にアクセス
  • 左手タブレット: http://192.168.1.100:3000/accompaniment にアクセス
  • 各タブレットが自動的に ws://192.168.1.100:8080 に WebSocket接続
3. 同期フェーズ
  • マスター制御パネルで「演奏開始」ボタン押下
  • ラズパイが全タブレットに同期開始信号送信
  • 指定時刻（例：100ms後）に全デバイスで同時にアニメーション開始
4. 演奏フェーズ
  • 各タブレットが同じタイムラインでノーツを表示
  • Web Audio API の高精度タイマーで同期維持
  • リアルタイムでテンポ調整・停止制御

🔗 コード間の連携

server.js (ラズパイ)

// HTTPサーバーでHTMLファイル配信
app.get('/melody', (req, res) => {
  res.sendFile('public/melody.html');
});

// WebSocketで同期制御
wss.on('connection', (ws) => {
  // タブレットとの通信処理
});

melody.html (タブレット)

// WebSocket接続
const ws = new WebSocket('ws://192.168.1.100:8080');

// 同期信号受信
ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  if (data.type === 'sync_start') {
    startAnimation(data.startTime);
  }
};

🎯 重要なポイント

  • 単一サーバー: ラズパイが HTTP + WebSocket の両方を担当
  • HTMLファイル配信: タブレットはラズパイから直接HTMLを取得
  • WebSocket常時接続: 演奏中はリアルタイム通信を維持
  • 共通タイムライン: Web Audio API の時刻を全デバイスで共有
